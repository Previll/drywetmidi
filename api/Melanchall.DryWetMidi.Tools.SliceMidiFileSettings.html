    PreserveTimes

    Gets or sets a value indicating whether original times of events should be saved or not. The default value is false.
    Declaration
    public bool PreserveTimes { get; set; }
    Property Value
    Type Description
    Boolean
    Remarks
    If false used, events will be moved to the start of a new file. If true used, events will be placed in new files at the same absolute times as in the input file.
    Examples

    Take a look at the following file with two track chunks: 

    ┌───────────────────────┐
│     E  E           E  │
└───────────────────────┘
┌───────────────────────┐
│   E      E   E      E │
└───────────────────────┘

    where each E symbol represents a MIDI event.

    Now let's imagine the file should be split as shown on the image below: 

    ┌────────────│──────────┐
│     E  E   │       E  │
└────────────│──────────┘
┌────────────│──────────┐
│   E      E │ E      E │
└────────────│──────────┘

    If PreserveTimes set to false (which is the default value), we'll get following two files: 

          File 1             File 2
··················  ················
· ┌────────────┐ ·  · ┌──────────┐ ·
· │     E  E   │ ·  · │       E  │ ·
· └────────────┘ ·  · └──────────┘ ·
· ┌────────────┐ ·  · ┌──────────┐ ·
· │   E      E │ ·  · │ E      E │ ·
· └────────────┘ ·  · └──────────┘ ·
··················  ················

    But if PreserveTimes set to true, we'll get following new files: 

          File 1
··················
· ┌────────────┐ ·
· │     E  E   │ ·
· └────────────┘ ·
· ┌────────────┐ ·
· │   E      E │ ·
· └────────────┘ ·
··················

            File 2
·····························
· ┌───────────────────────┐ ·
· │                    E  │ ·
· └───────────────────────┘ ·
· ┌───────────────────────┐ ·
· │              E      E │ ·
· └───────────────────────┘ ·
·····························

    So events will not be shifted to the start of a file and will keep their original absolute times.

    PreserveTrackChunks

    Gets or sets a value indicating whether track chunks in new files should correspond to those in the input file or not, so empty track chunks can be presented in new files. The default value is false.
    Declaration
    public bool PreserveTrackChunks { get; set; }
    Property Value
    Type Description
    Boolean
    Remarks

    If false used, track chunks without events will be removed from the result.

    Examples

    Take a look at the following file with two track chunks: 

    ┌───────────────────────┐
│     E  E           E  │
└───────────────────────┘
┌───────────────────────┐
│   E      E   E      E │
└───────────────────────┘

    where each E symbol represents a MIDI event.

    Now let's imagine the file should be split as shown on the image below: 

    ┌────────────│─────│────┐
│     E  E   │     │ E  │
└────────────│─────│────┘
┌────────────│─────│────┐
│   E      E │ E   │  E │
└────────────│─────│────┘

    If PreserveTrackChunks set to false (which is the default value), we'll get following three files: 

          File 1                      File 3
··················              ··········
· ┌────────────┐ ·              · ┌────┐ ·
· │     E  E   │ ·    File 2    · │ E  │ ·
· └────────────┘ ·  ··········  · └────┘ ·
· ┌────────────┐ ·  · ┌────┐ ·  · ┌────┐ ·
· │   E      E │ ·  · │ E  │ ·  · │  E │ ·
· └────────────┘ ·  · └────┘ ·  · └────┘ ·
··················  ··········  ··········

    But if PreserveTrackChunks set to true, we'll get following new files: 

          File 1          File 2      File 3
··················  ··········  ··········
· ┌────────────┐ ·  · ┌────┐ ·  · ┌────┐ ·
· │     E  E   │ ·  · │    │ ·  · │ E  │ ·
· └────────────┘ ·  · └────┘ ·  · └────┘ ·
· ┌────────────┐ ·  · ┌────┐ ·  · ┌────┐ ·
· │   E      E │ ·  · │ E  │ ·  · │  E │ ·
· └────────────┘ ·  · └────┘ ·  · └────┘ ·
··················  ··········  ··········

    So although first track chunk of the second new file is empty, it will be present in the file because it's present in the original one.

    SplitNotes

    Gets or sets a value indicating whether notes should be split in points of grid intersection or not. The default value is true.
    Declaration
    public bool SplitNotes { get; set; }
    Property Value
    Type Description
    Boolean
    Remarks

    false means notes treated as just Note On / Note Off events rather than note objects for true. Splitting notes produces new Note On / Note Off events at points of grid intersecting notes.

    Examples

    Take a look at the following file: 

    ┌───────────────────────┐
│       A.......B       │
└───────────────────────┘
┌───────────────────────┐
│   A...B          A..B │
└───────────────────────┘
┌───────────────────────┐
│     A............B    │
└───────────────────────┘

    where A and B mean Note On and Note Off events correspondingly. ... symbols are drawn to make it more clear what notes we have in each track chunk. Each block on the image represents a track chunk.

    Now let's imagine the file should be split by the line on image below: 

    ┌────────────│──────────┐
│       A    │  B       │
└────────────│──────────┘
┌────────────│──────────┐
│   A...B    │     A..B │
└────────────│──────────┘
┌────────────│──────────┐
│     A      │     B    │
└────────────│──────────┘

    If SplitNotes set to false, we'll get following two parts: 

          File 1             File 2
··················  ················
· ┌────────────┐ ·  · ┌──────────┐ ·
· │       A    │ ·  · │  B       │ ·
· └────────────┘ ·  · └──────────┘ ·
· ┌────────────┐ ·  · ┌──────────┐ ·
· │   A...B    │ ·  · │     A..B │ ·
· └────────────┘ ·  · └──────────┘ ·
· ┌────────────┐ ·  · ┌──────────┐ ·
· │     A      │ ·  · │     B    │ ·
· └────────────┘ ·  · └──────────┘ ·
··················  ················

    But if SplitNotes set to true, we'll get following new files: 

          File 1             File 2
··················  ················
· ┌────────────┐ ·  · ┌──────────┐ ·
· │       A....B ·  · A..B       │ ·
· └────────────┘ ·  · └──────────┘ ·
· ┌────────────┐ ·  · ┌──────────┐ ·
· │   A...B    │ ·  · │     A..B │ ·
· └────────────┘ ·  · └──────────┘ ·
· ┌────────────┐ ·  · ┌──────────┐ ·
· │     A......B ·  · A.....B    │ ·
· └────────────┘ ·  · └──────────┘ ·
··················  ················

    So we have new Note On / Note Off events at points of split which gives us new notes.
